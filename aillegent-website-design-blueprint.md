## 3. Shared visual and interaction direction

### Visual language

- **Mood:** Premium, technical, precise, and human—not generic “futuristic AI.”
- **Palette:** Deep graphite or midnight base with electric blue/teal for active states. Use soft neutral surfaces for reading-heavy sections.
- **Typography:** Modern sans-serif, generous spacing, high contrast, and short headings. Avoid dense all-caps text.
- **Imagery:** Product interfaces, clear architecture visualisations, abstract data or infrastructure details. Do not use generic teams-in-office stock photography as the primary visual.
- **Layout:** Wide editorial sections, structured capability cards, concise process diagrams, and generous whitespace.

### Motion principles

- Motion must show progress, connection, or state change—not decorate empty space.
- Use 500–700 ms transitions for product interfaces, cards, and cycle rotation.
- Respect prefers-reduced-motion; the same information must work with no animation.
- Do not use continuous background spinning or repeated “AI particle” loops.

### Signature interaction: rotating engagement cycle

Use this in the Home and How We Work pages.

1. Visitors select a numbered stage, a stage label, or Next stage.
2. The outer ring rotates until the selected stage reaches the top.
3. The centre shows the stage name and sequence number.
4. The supporting panel changes to the stage explanation and outcome.
5. The selected stage is also announced to keyboard and screen-reader users.

| Cycle | Stages |
|---|---|
| Delivery cycle | Discover → Architect → Build → Launch → Validate → Optimise |
| Growth journey | Quick MVP → Startup Growth → Enterprise Production |

1. **Business requirement scenario breakdown** — Convert real business needs into clear testable scenarios.
2. **Feature-to-scenario mapping** — Ensure each product feature has meaningful coverage.
3. **Testing environment → UAT → production** — Maintain a disciplined path from internal testing to user validation and live release.

5. **Web automation testing** — Production-scale automated browser testing for critical user journeys.
6. **API testing automation** — Verify service contracts, integrations, and backend behaviour.
7. **Performance testing** — Understand how the platform behaves under realistic demand.
